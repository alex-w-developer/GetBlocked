# Recommended GitHub Labels

Use labels to make it easy for contributors to find small, useful PRs.

- `good first issue`: Small, well-scoped task suitable for new contributors.
- `help wanted`: Maintainers welcome outside help.
- `PR welcome`: A pull request is preferred over a suggestion-only issue.
- `tracker-domain`: Adds, removes, or researches tracker catalog domains.
- `broken-site`: Reports or fixes website breakage caused by blocking.
- `documentation`: README, docs, comments, or contributor guidance.
- `tests`: Test fixtures, scripts, CI, or validation improvements.
- `UI`: Popup layout, copy, styling, or accessibility polish.
- `MV3`: Manifest V3, DNR, service worker, or Chrome extension architecture.
- `research`: Needs investigation before a rule or product change.
- `needs reproduction`: Needs clear steps before a fix can be reviewed.
- `safe expansion`: Adds tracker coverage with low expected breakage risk.
- `do not add: high breakage`: Domain or idea is too broad or risky for the blocker catalog.

For contributor growth, pair `good first issue` with `PR welcome` whenever the issue has clear files to edit and commands to run.
